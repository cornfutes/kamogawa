Platform Overview

UI-as-a-Service

We provide the best-in-class web console for all your cloud management needs. We are not constrained by organizational challenges that prevent cloud platforms from delivering software that make sense.

In fact, cloud vendors are disincentivized from facilitating multi-cloud adoption. The full potential of cloud-native has yet to be realized by organizations, ironically inhibited by these utility services. We believe we can make money without being evil. This translates to making your cloud operation easier and more cost-effective.

Our edge is in what we lack:

  • Bloated 20MB+ bundles of JavaScript/CSS
  • Cluttered UIs mirroring the cloud vendors' org structure rather than what is applicable to your workflow
  • Uninspired material design
  • Irrelevant content for a theoretical frankenstein client
Our product roadmap is driven by utility. Support for web browsers that are NOT desktop Chrome is also important.

One API

When you need more expressive queries or automation at scale, we have a GraphQL API. A single API to integrate and familiarize with, rather than a dozen quirky ones.

Incentive structures at cloud vendors lead to APIs provided that are half-baked at launch, then in an ambiguous state of maintenance. SDKs and libraries are inconsistently implemented. They are subject to idiosyncrasies at the whim of individual developers, especially in ecosystems that are $NOT_JAVA. Autogen is not a panacea, either.

Documentation is the worst. For example, there are 2.5x sets of documentation alluding to authentication with GCP APIs. In the end, you have to dig through source code to uncover a specific parameter, how to pass it, and finally correlate this to your specific library / API version. This is a failed abstraction that puts the burden of fragile integration code on your team.

We manage the complexities of vendoring and versioning issues, so you have reliability and predictability. We are uniquely positioned for economy-of-scale in regard to smoothing out API quirks once-and-for-all. Your software engineering woes are literally our business. Payment processors existed before Stripe, and we're kind of like Stripe, but for Cloud vendors rather than banks.
